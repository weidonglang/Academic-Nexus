# Academic-Nexus v2.0.1 - AI, Docker and Frontend UX Fixes

This patch release improves AI search reliability, Docker deployment documentation, AI model registry consistency, frontend interaction completeness, AI Chat controllability, and project branding.

## Branding

This release updates the project display name to **EduNexus AI**.

EduNexus AI is an AI-powered academic management platform for university teaching administration scenarios, covering student, teacher and admin workflows, AI assistant, RAG, natural-language SQL, Redis-backed course selection and Docker-based deployment.

Chinese name: **智教中枢**.

## Fixed

- Clarified SearXNG real search configuration in Docker networks.
- Clarified Ollama enablement for Docker ai-service deployment.
- Fixed web-search grounding so local model answers reliably use returned search references.
- Fixed AI model registry state constraints for default, disabled, DOWN and deleted models.
- Fixed top global search interaction.
- Fixed dashboard quick-app settings interaction.

## Improved

- Improved AI Chat frontend status display, loading feedback, retry/copy actions, error messages, and search/safety visibility.
- Added AI thinking mode switch: Auto / On / Off.
- Added thinkingMode propagation from frontend to main service and ai-service.
- Updated README, frontend title, login page, shell footer, Docker docs, AI search docs and AI prompts to use EduNexus AI.
- Kept internal package names, API paths, historical Docker service names and repository path unchanged for compatibility.
