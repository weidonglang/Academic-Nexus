# v1.4.1 Open Issues Closure Report

Updated: 2026-06-27

This report records the v1.4.1 stabilization work for open issues #76-#101. The focus is bug closure and business-link repair on top of v1.4.0, not adding another large feature wave.

## Fixed Or Improved

| Area | Result |
| --- | --- |
| #76 AI model selection | Selected chat model now travels through frontend, main system, Feign/HTTP payload, and ai-service. AI call logs record selected/actual/fallback fields. |
| #77 Sensitive words | Word list and moderation logs load independently; backend returns stable empty pages when governance tables drift. |
| #78 Load-test panel | Parser supports `data[]`, `data.records[]`, `id`, `offeringId`, term filtering, and explicit diagnostics. |
| #90 Auth session | Refresh, logout, disabled-user rejection, and admin token revocation are implemented. |
| #91 Authenticated downloads | Attachment preview/download, DB CSV, archive CSV, and batch task report downloads use authenticated axios blob requests. |
| #93 Data masking | Database browser and schema service mask broader sensitive fields including password, token, phone, email, address, identity, key, hash, and salt columns. |
| #94 File management | Admin and student attachment file access is constrained to upload root and writes audit logs for upload/download/preview/delete. |
| #95 Redis stock | Redis prewarm and consistency repair remain DB-authoritative; prewarm is now audited. |
| #96 System health | Health center reports runtime profile/port, MySQL, Redis, Nacos config, AI, upload path, Flyway, demo data, release zip, and JVM. |
| #97 DB browser | Schema, index, foreign-key, dashboard, tree, and history loads no longer fail the entire page when one section fails. |
| #98 Attachment upload state | Students cannot add materials after a status-change application is approved/rejected/canceled. |
| #99 Current term | Course selection and selected-course APIs use `system_config.current_term`, request term, latest offering term, or date inference instead of a hardcoded term. |
| #100 Schedule parsing | Personal schedule detects abnormal schedule text and shows it separately instead of placing it incorrectly. |
| #101 Grab validation | `offeringId` is required with a readable validation message. |
| Grade/exam closure | Locked grades cannot be changed; grade point is range-checked; exam create/update/delete sends notifications to selected students. |
| Menus/routes | Frontend fallback admin menus now include all implemented v1.4 admin pages. |
| Batch task reports | Batch task center can download a CSV report for every task row. |

## Remaining Scope To Track Separately

These items are not safe to mark fully closed by this stabilization pass:

| Area | Reason |
| --- | --- |
| Full two-stage batch user/course/offering import | Current code has basic CSV/course paths and batch task records, but not the complete preview/commit/report UI/API contract requested in #79/#80. |
| Batch review for all application pages | Single-review paths are stable; full multi-select partial-success reporting still needs a focused implementation pass. |
| Advanced notice targeting by grade/major/class/offering | Role targeting and stats are stable; fine-grained recipient preview is not fully implemented. |
| Teacher readonly awareness for applications | Teacher ownership pages remain protected, but the requested readonly status-change/application summary was not added in this pass. |
