# Academic-Nexus v1.3.0 Issue Closure Report

Updated: 2026-06-27

This report records the v1.3.0 closure scope for GitHub issues #39-#59. The release is a bugfix, QA, and acceptance release on top of v1.2. It intentionally focuses on existing business paths instead of adding new large modules.

## Scope Rules

- Fix existing 500 errors, no-response UI states, permission drift, and AI fallback messages.
- Verify the current student, teacher, admin, AI, Redis, Spring Cloud, Docker, and release-package paths.
- Do not implement new full modules for areas without existing product entry points, including graduation thesis workflow, course materials/homework, leave/attendance, and internship/practice workflow. These are documented as out of scope for v1.3.0.

## Code Fixes

| Area | Fix |
| --- | --- |
| Authentication | Added safe default token TTL values so login does not fail with a 500 when security TTL properties are absent in a test or fallback environment. |
| Multi-role users | Ordered returned role codes as ADMIN, TEACHER, STUDENT, then custom roles, so the header and permission decisions have stable priority. |
| AI service fallback | Converted Spring Cloud LoadBalancer and connection failures into a readable `academic-ai-service` fallback message instead of raw framework errors. |
| AI safety config | Added migration-drift tolerant reads and moderation logging fallback so missing governance tables do not break page loading with a 500. |
| Evaluation details | Added user feedback when switching to all evaluation records and surfaced load errors through Element Plus messages. |
| Notice, registration, status change pages | Added visible load failure messages instead of silent no-response states. |
| AI admin page | Changed initial loading to partial success with warnings and added save/test/toggle error messages for model, search, and safety config actions. |

## Automated Regression Coverage

Added `QaClosureHttpRegressionTests` to cover the v1.3.0 hot paths through a real random-port Spring Boot server:

- #39: student status-change submit, registration submit, admin notice publish, AI safety config GET/PUT return non-500 responses.
- #41: anonymous, student, and cross-role forbidden access behave as 401/403; multi-role login returns ADMIN first.
- #44: AI status returns the main-system fallback mode and readable error when `academic-ai-service` is unavailable through service discovery.
